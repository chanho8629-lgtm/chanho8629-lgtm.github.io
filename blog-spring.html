Overview

Spring Boot 3.5 + Java 17 기반 RESTful API 백엔드 설계

BIDEO(영상 경매 플랫폼)와 Clausion(AI 기반 교육 플랫폼) 두 프로젝트에서 Spring Boot를 직접 적용했습니다. 단순한 CRUD를 넘어 인증/인가, 비동기 메시징, DB 스키마 관리, API 문서화까지 실제 서비스 수준의 백엔드 구조를 설계하고 구현했습니다. Java 17의 Record, Sealed Class, Pattern Matching 등 최신 문법과 Spring Boot 3.x의 개선된 자동 구성을 활용하여 가독성과 유지보수성을 높이는 데 초점을 맞췄습니다.

Core Topics

핵심 학습 주제

🔐
Spring Security + JWT
인증/인가 파이프라인을 SecurityFilterChain 기반으로 설계했습니다. Access Token(15분)과 Refresh Token(7일) 전략을 적용해 탈취 위험을 최소화했고, Redis에 Refresh Token을 저장하여 서버 측 만료 처리와 Token Rotation을 구현했습니다. CORS는 CorsConfigurationSource 빈으로 도메인별 허용 정책을 명시적으로 관리했습니다.
🗄️
JPA vs MyBatis — ORM 비교 적용
연관관계가 복잡한 도메인(사용자-영상-경매-입찰)은 JPA의 엔티티 그래프와 Fetch Join으로 처리했고, 통계 쿼리나 대량 데이터 배치 처리처럼 SQL 제어가 중요한 경우 MyBatis를 선택했습니다. 두 기술을 같은 프로젝트에서 병행 사용하며 각 ORM의 장단점을 체감했습니다.
🐇
RabbitMQ — 메시지 큐 비동기 처리
영상 업로드 후처리(인코딩, 썸네일 생성), 입찰 알림 발송, 이메일 전송을 RabbitMQ 메시지 큐로 분리했습니다. Exchange/Queue/Binding 구조를 직접 설계하고, Dead Letter Queue로 처리 실패 메시지를 추적했습니다. API 응답 속도가 평균 340ms에서 80ms로 단축되는 효과를 확인했습니다.
⚡
Spring WebFlux — 비동기 논블로킹
AI API(OpenAI, Claude)처럼 응답이 느린 외부 API를 호출할 때 WebFlux의 WebClient를 활용해 스레드 블로킹 없이 처리했습니다. Mono/Flux 타입으로 리액티브 파이프라인을 구성하고, SSE(Server-Sent Events)로 AI 스트리밍 응답을 클라이언트에 전달하는 구조를 구현했습니다.
🔄
Flyway — DB 마이그레이션 버전 관리
팀 협업 환경에서 스키마 변경을 SQL 파일로 버전 관리했습니다. V1__init.sql부터 순번을 부여하여 어떤 환경에서도 동일한 스키마를 재현할 수 있게 했습니다. 롤백 전략(Undo Migration)과 체크섬 검증으로 배포 사고를 예방했습니다.
📝
Swagger / SpringDoc — API 문서 자동화
springdoc-openapi로 컨트롤러 어노테이션에서 API 스펙을 자동 생성했습니다. @Operation, @ApiResponse, @Schema로 상세 설명을 추가하여 프론트엔드 팀이 별도 문서 없이도 API를 파악하고 테스트할 수 있는 환경을 만들었습니다.
Code Concepts

핵심 코드 개념

JWT 토큰 생성 — JwtTokenProvider.java
// JWT 토큰 생성 예시 @Component public class JwtTokenProvider { @Value("${jwt.expiration}") private long expiration; private final SecretKey secretKey = Keys.secretKeyFor(SignatureAlgorithm.HS256); public String generateToken(UserDetails user) { return Jwts.builder() .setSubject(user.getUsername()) .setIssuedAt(new Date()) .setExpiration(new Date(System.currentTimeMillis() + expiration)) .signWith(secretKey, SignatureAlgorithm.HS256) .compact(); } public boolean validateToken(String token) { try { Jwts.parserBuilder().setSigningKey(secretKey).build().parseClaimsJws(token); return true; } catch (JwtException e) { return false; } } }
RabbitMQ 메시지 발행 — VideoUploadService.java
// RabbitMQ 메시지 발행 @Service @RequiredArgsConstructor public class VideoUploadService { private final RabbitTemplate rabbitTemplate; private final S3Client s3Client; public UploadResponse processUpload(MultipartFile file, Long userId) { String fileKey = uploadToS3(file, userId); // 비동기 인코딩 작업 발행 rabbitTemplate.convertAndSend( "video.exchange", "upload.key", new VideoUploadMessage(fileKey, userId) ); return new UploadResponse(fileKey, "PROCESSING"); } } // 메시지 수신 처리 @Component public class VideoUploadConsumer { @RabbitListener(queues = "video.upload.queue") public void handleUpload(VideoUploadMessage msg) { // 인코딩, 썸네일 생성, 메타데이터 저장 videoEncoder.encode(msg.getFileKey()); thumbnailGenerator.generate(msg.getFileKey()); } }
Spring Security 설정 — SecurityConfig.java
@Configuration @EnableWebSecurity @RequiredArgsConstructor public class SecurityConfig { private final JwtTokenProvider jwtProvider; @Bean public SecurityFilterChain filterChain(HttpSecurity http) throws Exception { return http .csrf(AbstractHttpConfigurer::disable) .sessionManagement(s -> s.sessionCreationPolicy(STATELESS)) .authorizeHttpRequests(auth -> auth .requestMatchers("/api/auth/**", "/api/public/**").permitAll() .anyRequest().authenticated() ) .addFilterBefore(new JwtAuthFilter(jwtProvider), UsernamePasswordAuthenticationFilter.class) .build(); } }
Real-World Application

실제 적용 사례

🎬 BIDEO — 실시간 입찰 API

WebSocket + STOMP 기반 실시간 입찰 처리 API 설계. 동시 입찰 충돌 방지를 위해 Redis 분산 락(Redisson)을 적용했습니다. 경매 종료 시 RabbitMQ로 낙찰 알림과 정산 프로세스를 비동기로 처리했습니다.

🤖 Clausion — AI 챗봇 엔드포인트

Spring WebFlux + WebClient로 OpenAI Streaming API를 연동했습니다. SSE(Server-Sent Events)로 토큰이 생성되는 즉시 클라이언트에 스트리밍하여 체감 응답 속도를 크게 개선했습니다.

Lessons Learned

개발하며 배운 것들

  • Spring Security 설정의 복잡성 — 필터 체인 순서, 예외 처리 흐름, SecurityContext 전파 방식을 한 번씩 직접 트러블슈팅하며 동작 원리를 완전히 이해했습니다.
  • JPA N+1 문제 해결 — 연관 엔티티를 Lazy Loading 기본으로 설정하고, 필요한 경우에만 Fetch Join(@EntityGraph)을 적용하는 패턴을 정착시켰습니다. @BatchSize를 컬렉션에 적용해 IN 쿼리로 최적화했습니다.
  • 비동기 처리의 중요성 — 동기 방식으로 AI API를 호출하면 스레드가 수십 초씩 블로킹됩니다. WebFlux + Virtual Thread(Java 21)를 통해 처리량을 3배 이상 향상시킨 경험을 했습니다.
  • Flyway 마이그레이션 규칙 — 이미 적용된 마이그레이션 파일을 수정하면 체크섬 오류가 발생합니다. 절대로 기존 파일을 수정하지 않고 새 버전 파일을 추가하는 규칙을 팀 내에서 정착시켰습니다.
  • 트랜잭션 경계 설계 — @Transactional 범위가 너무 넓으면 DB 커넥션을 오래 점유하고, 너무 좁으면 데이터 정합성이 깨집니다. 서비스 레이어 단위로 트랜잭션 경계를 명확히 설계하는 습관을 들였습니다.
