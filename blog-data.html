Overview

Pandas · NumPy · Scikit-learn으로 데이터 분석과 ML 모델 구현

BIDEO 플랫폼에서 수집된 사용자 행동 데이터(입찰, 영상 조회, 구매 이력)를 분석하여 비즈니스 인사이트를 도출했습니다. RFM 분석으로 사용자를 세그멘테이션하고, 협업 필터링으로 개인화 추천을 구현했습니다. 경매 낙찰가 예측 모델(R² = 0.82)과 사용자 이탈 예측 모델(Precision 0.78)을 개발하여 플랫폼 운영 의사결정에 실제로 활용했습니다.

Core Topics

핵심 학습 주제

🐼
Pandas — DataFrame 조작과 시계열 분석
결측치 처리(fillna, interpolate), 이상치 제거(IQR 방법), 그룹 집계(groupby + agg)를 활용하여 원시 데이터를 분석 가능한 형태로 가공했습니다. resample로 일/주/월 단위 시계열 집계를 처리하고, rolling window로 이동평균 추세를 계산했습니다. merge, pivot_table, melt로 복잡한 데이터 변환을 처리했습니다.
🔢
NumPy — 벡터화 연산과 행렬 계산
Python 반복문 대신 NumPy 벡터화 연산으로 처리 속도를 100배 이상 향상시켰습니다. 브로드캐스팅으로 형태가 다른 배열 간 연산을 처리하고, np.linalg로 선형대수 계산을 수행했습니다. 임베딩 벡터 간 코사인 유사도 행렬을 NumPy 행렬 곱으로 효율적으로 계산했습니다.
📈
RFM 분석 — 사용자 세그멘테이션
Recency(최근성), Frequency(빈도), Monetary(금액) 세 지표로 사용자 가치를 정량화했습니다. qcut으로 각 지표를 4분위로 나눠 RFM 점수를 계산하고, K-Means 클러스터링으로 VIP/활성/휴면/이탈 위험 4개 세그먼트로 분류했습니다. 세그먼트별 맞춤 마케팅 전략 수립에 활용했습니다.
🌲
의사결정 트리 — 분류와 회귀
사용자 이탈 예측(분류)과 낙찰가 예측(회귀)에 의사결정 트리를 적용했습니다. max_depth, min_samples_split으로 과적합을 방지하고, feature_importances_로 중요 변수를 파악했습니다. 단일 트리의 한계를 극복하기 위해 Random Forest와 Gradient Boosting을 비교 적용했습니다.
🤝
협업 필터링 — 개인화 추천
사용자-영상 상호작용 행렬을 구성하고 코사인 유사도로 유사 사용자를 찾아 영상을 추천했습니다. Item-based 협업 필터링으로 유사 영상을 추천하고, Matrix Factorization(SVD)으로 희소 행렬 문제를 해결했습니다. 콜드 스타트 문제는 콘텐츠 기반 필터링(TF-IDF)으로 보완했습니다.
📉
선형 회귀 — 경매 낙찰가 예측
영상 조회수, 좋아요 수, 게시일, 카테고리, 판매자 평점 등 다변수로 낙찰가를 예측했습니다. Ridge/Lasso 정규화로 다중공선성을 처리하고, PolynomialFeatures로 비선형 관계를 포착했습니다. 교차 검증(KFold)으로 모델 일반화 성능을 검증했습니다.

핵심 코드 개념

RFM 분석 — 사용자 세그멘테이션
# RFM 분석으로 사용자 등급화 import pandas as pd from datetime import datetime from sklearn.preprocessing import StandardScaler from sklearn.cluster import KMeans # 1. RFM 지표 계산 snapshot_date = datetime.now() rfm = df.groupby('user_id').agg({ 'created_at': lambda x: (snapshot_date - x.max()).days, # Recency 'bid_id': 'count', # Frequency 'bid_amount': 'sum' # Monetary }).rename(columns={'created_at': 'R', 'bid_id': 'F', 'bid_amount': 'M'}) # 2. 4분위 점수 부여 (R은 낮을수록 좋아서 역순) rfm['R_score'] = pd.qcut(rfm['R'], 4, labels=[4,3,2,1]).astype(int) rfm['F_score'] = pd.qcut(rfm['F'], 4, labels=[1,2,3,4]).astype(int) rfm['M_score'] = pd.qcut(rfm['M'], 4, labels=[1,2,3,4]).astype(int) rfm['RFM_score'] = rfm['R_score'] + rfm['F_score'] + rfm['M_score'] # 3. K-Means 클러스터링으로 세그먼트 분류 scaler = StandardScaler() rfm_scaled = scaler.fit_transform(rfm[['R', 'F', 'M']]) rfm['segment'] = KMeans(n_clusters=4, random_state=42).fit_predict(rfm_scaled) # 세그먼트별 평균 확인 print(rfm.groupby('segment')[['R','F','M']].mean())
콘텐츠 기반 영상 유사도 추천
# TF-IDF 기반 영상 유사도 추천 from sklearn.feature_extraction.text import TfidfVectorizer from sklearn.metrics.pairwise import cosine_similarity import numpy as np # 영상 설명 텍스트로 TF-IDF 행렬 생성 tfidf = TfidfVectorizer( max_features=5000, ngram_range=(1, 2), stop_words='english' ) tfidf_matrix = tfidf.fit_transform(videos['description']) # 코사인 유사도 행렬 (NxN) sim_matrix = cosine_similarity(tfidf_matrix) # 비디오 ID → 인덱스 매핑 video_id_to_idx = {vid: idx for idx, vid in enumerate(videos['id'])} def recommend(video_id: int, n: int = 5) -> list: idx = video_id_to_idx[video_id] scores = list(enumerate(sim_matrix[idx])) # 자기 자신 제외, 유사도 내림차순 정렬 scores = sorted(scores, key=lambda x: x[1], reverse=True)[1:n+1] return [videos.iloc[i[0]]['id'] for i in scores]
낙찰가 예측 — 다변수 회귀 모델
# 경매 낙찰가 예측 모델 from sklearn.linear_model import Ridge from sklearn.model_selection import cross_val_score, KFold from sklearn.preprocessing import StandardScaler, OneHotEncoder from sklearn.pipeline import Pipeline from sklearn.compose import ColumnTransformer import numpy as np # 특성 전처리 파이프라인 numeric_features = ['view_count', 'like_count', 'seller_rating', 'video_duration'] categorical_features = ['category', 'quality'] preprocessor = ColumnTransformer([ ('num', StandardScaler(), numeric_features), ('cat', OneHotEncoder(handle_unknown='ignore'), categorical_features) ]) # Ridge 회귀 파이프라인 model = Pipeline([ ('prep', preprocessor), ('regressor', Ridge(alpha=1.0)) ]) # 교차 검증으로 성능 평가 kf = KFold(n_splits=5, shuffle=True, random_state=42) r2_scores = cross_val_score(model, X, y, cv=kf, scoring='r2') print(f"R² Mean: {r2_scores.mean():.3f} ± {r2_scores.std():.3f}") # R² Mean: 0.820 ± 0.031

Lessons Learned

개발하며 배운 것들

  • 데이터 전처리의 중요성 — 모델 선택보다 데이터 품질이 훨씬 중요합니다. 결측치, 이상치, 스케일 차이를 제대로 처리한 후 단순 선형 회귀만으로도 R² 0.82를 달성했습니다.
  • 교차 검증 필수 — 단순 train/test 분리는 데이터 분할 운에 따라 성능이 달라집니다. K-Fold 교차 검증으로 모델의 실제 일반화 성능을 신뢰할 수 있게 측정했습니다.
  • 오버피팅 방지 — 학습 데이터 정확도는 높지만 검증 데이터에서 낮으면 오버피팅입니다. max_depth, min_samples, 정규화(Ridge/Lasso) 하이퍼파라미터 튜닝으로 해결했습니다.
  • 특성 공학의 힘 — 원시 특성보다 파생 특성(영상 나이, 판매자 경매 승률, 카테고리별 평균 낙찰가)이 모델 성능을 더 크게 향상시켰습니다. 도메인 지식이 ML 성능을 결정합니다.
  • 평가 지표 선택 — 이탈 예측에서 Accuracy만 보면 "모두 안 이탈"로 예측해도 70%가 나옵니다. 불균형 데이터에서는 Precision, Recall, F1을 함께 봐야 합니다.
