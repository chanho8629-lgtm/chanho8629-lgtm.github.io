Overview

프로젝트 개요

Role

담당 역할

Features

핵심 기능

🔴

실시간 입찰 시스템

WebSocket + STOMP 프로토콜로 경매 상태를 실시간으로 동기화합니다. Redis Pub/Sub을 활용해 다중 서버 환경에서도 모든 클라이언트에 일관된 입찰 현황을 전달합니다.

💳

안전한 결제 처리

PG사 연동을 통한 안전한 결제 흐름을 설계했습니다. 낙찰 후 결제 실패 시 트랜잭션 롤백 전략으로 데이터 정합성을 보장하며, 환불 로직까지 완전히 구현했습니다.

📁

영상 업로드 파이프라인

AWS S3 멀티파트 업로드로 대용량 영상을 분할 전송합니다. RabbitMQ 메시지 큐를 통한 비동기 처리로 업로드 중 서버 부하를 최소화하고, 파일 크기·포맷 검증을 적용했습니다.

🤖

AI 영상 추천

LangChain + RAG 파이프라인으로 영상 메타데이터를 벡터화하고, ChromaDB에 저장합니다. 유사도 검색으로 사용자 취향에 맞는 영상을 추천하는 지능형 검색 시스템을 구현했습니다.

📊

낙찰가 예측 모델

Scikit-learn 회귀 모델(Random Forest, Gradient Boosting)로 영상 속성과 과거 낙찰 데이터를 분석해 예상 낙찰가를 제공합니다. 판매자의 합리적인 시작가 설정을 도왔습니다.

👤

인증 / 인가 시스템

Spring Security + JWT 기반 stateless 인증을 구현했습니다. Access Token 만료 시 Refresh Token으로 자동 갱신하는 전략을 적용해 보안성과 사용자 편의를 동시에 확보했습니다.

AI & Analytics

AI & 데이터 분석

데이터 기반 의사결정 시스템

  • 사용자 행동 데이터 수집 및 Pandas / NumPy 기반 탐색적 분석(EDA) 수행
  • 영상 제목·설명·태그 임베딩으로 콘텐츠 유사도 분석 → 개인화 추천 시스템 구현

  • LangChain RAG 파이프라인(BIDEO-RAG) — 영상 Q&A 챗봇 및 검색 증강 생성
  • 낙찰가 예측 회귀 모델 구축 및 검증 (task_video.ipynb 노트북으로 전 과정 문서화)

Architecture

아키텍처 플로우

01
클라이언트 요청 처리
React / 모바일 클라이언트 → Spring Boot REST API → JWT 인증 필터 → 비즈니스 로직 → PostgreSQL 데이터 영속화
02
실시간 입찰 처리
WebSocket / STOMP 연결 → 입찰 이벤트 발행 → Redis Pub/Sub 브로드캐스트 → 모든 클라이언트 실시간 동기화
03
영상 파일 비동기 처리
멀티파트 업로드 요청 → RabbitMQ 메시지 큐 → Consumer 비동기 처리 → AWS S3 업로드 완료 → DB 상태 업데이트
04
AI 추천 파이프라인
추천 요청 → FastAPI 마이크로서비스 → LangChain 임베딩 생성 → ChromaDB 벡터 검색 → 유사 영상 Top-K 반환
