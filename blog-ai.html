Overview

LangChain + OpenAI API로 RAG 파이프라인 구축

단순히 OpenAI API를 호출하는 것을 넘어, 도메인 특화 지식을 LLM에 주입하는 RAG(Retrieval-Augmented Generation) 파이프라인을 설계했습니다. FastAPI로 AI 모델을 REST 엔드포인트로 노출하여 Spring Boot 백엔드와 연동했습니다. BIDEO에서는 영상 메타데이터를 임베딩하여 유사 영상 추천·가격 예측에 활용했고, Clausion에서는 강의 자료를 벡터화하여 학습 맥락을 인식하는 AI 튜터를 구현했습니다.

문서 로드 → 텍스트 청킹 → 임베딩 생성 → 벡터 저장 (ChromaDB) → 유사도 검색 → LLM 답변 생성
Core Topics

핵심 학습 주제

⚡
FastAPI — Python 비동기 웹 프레임워크
Pydantic 모델로 요청/응답 타입을 정의하면 자동으로 OpenAPI 문서가 생성됩니다. async/await 기반으로 동시에 여러 AI API 호출을 병렬 처리하여 응답 시간을 단축했습니다. BackgroundTasks로 느린 작업(임베딩 인덱싱)을 응답 후 비동기로 처리하는 패턴을 적용했습니다.
🦜
LangChain — LLM 오케스트레이션
여러 LLM, 도구, 메모리를 조합하는 체인과 에이전트를 구성했습니다. ConversationBufferMemory로 대화 맥락을 유지하고, Tool을 정의하여 LLM이 검색·계산·API 호출을 스스로 선택하도록 했습니다. LangSmith로 체인 실행 과정을 추적하고 디버깅했습니다.
🔍
RAG 파이프라인 — 검색 증강 생성
문서를 RecursiveCharacterTextSplitter로 청킹하고, OpenAI text-embedding-3-small로 임베딩을 생성합니다. 청킹 크기(chunk_size)와 중복(overlap)을 조정하며 검색 정확도와 비용의 균형을 맞췄습니다. Hybrid Search(키워드 + 벡터)로 검색 정확도를 향상시켰습니다.
🗃️
ChromaDB — 벡터 데이터베이스
임베딩 벡터를 저장하고 코사인 유사도 기반으로 유사 문서를 검색합니다. Collection에 메타데이터(카테고리, 날짜, 작성자)를 함께 저장하여 필터링 검색을 지원했습니다. Persistent Client로 서버 재시작 후에도 인덱스를 유지하고, Docker로 독립 서버로 배포했습니다.
🎨
멀티모달 — GPT-4V 이미지 + 텍스트 분석
BIDEO의 영상 썸네일과 제목을 함께 분석하여 카테고리를 자동 태깅했습니다. base64로 인코딩한 이미지를 메시지에 포함하고, GPT-4o Vision으로 이미지 내용을 설명하고 분류했습니다. 멀티모달 입력을 표준화하는 어댑터 클래스를 설계하여 모델 변경 시 영향 범위를 최소화했습니다.
🔗
LCEL — LangChain Expression Language
파이프(|) 연산자로 프롬프트 → LLM → 파서 → 후처리를 선언적으로 연결합니다. 각 단계가 Runnable 인터페이스를 구현하므로 병렬 실행(RunnableParallel), 분기 처리, 스트리밍이 간단합니다. invoke/ainvoke/stream/astream으로 동기·비동기·스트리밍 실행을 동일한 인터페이스로 처리했습니다.

RAG 파이프라인 — RetrievalQA 구성
# RAG 파이프라인 구성 from langchain_openai import OpenAIEmbeddings, ChatOpenAI from langchain_chroma import Chroma from langchain.chains import RetrievalQA from langchain.text_splitter import RecursiveCharacterTextSplitter # 1. 문서 청킹 splitter = RecursiveCharacterTextSplitter( chunk_size=500, chunk_overlap=50, separators=["\n\n", "\n", " "] ) chunks = splitter.split_documents(documents) # 2. 벡터스토어 생성 vectorstore = Chroma.from_documents( documents=chunks, embedding=OpenAIEmbeddings(model="text-embedding-3-small"), persist_directory="./chroma_db" ) # 3. QA 체인 구성 qa_chain = RetrievalQA.from_chain_type( llm=ChatOpenAI(model="gpt-4o"), retriever=vectorstore.as_retriever(search_kwargs={"k": 4}), chain_type="stuff", return_source_documents=True )
LCEL 스트리밍 파이프라인 — Clausion 챗봇
# LCEL 스트리밍 챗봇 파이프라인 from langchain_core.prompts import ChatPromptTemplate from langchain_core.runnables import RunnablePassthrough from fastapi.responses import StreamingResponse prompt = ChatPromptTemplate.from_template(""" You are an expert tutor for the following course material. Context: {context} Question: {question} Answer in Korean, concisely and accurately. """) chain = ( {"context": retriever, "question": RunnablePassthrough()} | prompt | llm | StrOutputParser() ) @app.post("/chat/stream") async def chat_stream(request: ChatRequest): async def generate(): async for chunk in chain.astream(request.message): yield f"data: {chunk}\n\n" return StreamingResponse(generate(), media_type="text/event-stream")
Case Study — BIDEO

BIDEO RAG 추천 시스템

🎬 영상 메타데이터 임베딩

영상 제목, 설명, 태그, 카테고리를 하나의 문서로 구성하고 OpenAI Embeddings로 벡터화했습니다. 신규 영상 업로드 시 RabbitMQ 이벤트를 수신하여 자동으로 ChromaDB에 인덱싱합니다.

💰 경매 낙찰가 예측

유사한 영상의 과거 낙찰 데이터를 검색하고, LLM이 시장 트렌드와 영상 특성을 종합하여 예상 낙찰가 범위를 제시합니다. 판매자가 시작가를 설정할 때 참고 지표로 활용됩니다.

Case Study — Clausion

Clausion AI 튜터 챗봇

🤖 학습 맥락 인식 AI 튜터

수강 중인 강의 자료(PDF, 영상 자막, 코드 파일)를 ChromaDB에 인덱싱하고, 학습자의 질문에 강의 내용 기반으로 정확한 답변을 제공합니다. 대화 히스토리를 ConversationBufferWindowMemory에 저장하여 이전 질문의 맥락을 이어받습니다. 코드 질문 시 CodeInterpreter Tool을 호출하여 실행 결과와 함께 설명합니다.

Lessons Learned

개발하며 배운 것들

  • 프롬프트 엔지니어링 — 프롬프트 하나가 응답 품질을 결정합니다. Few-shot 예시 추가, 역할 부여(System Prompt), 출력 형식 지정으로 응답 일관성을 크게 향상시켰습니다.
  • 토큰 비용 최적화 — text-embedding-3-small이 large보다 5배 저렴하면서 성능 차이가 미미합니다. 불필요한 컨텍스트를 줄이고 청킹 크기를 최적화하여 API 비용을 60% 절감했습니다.
  • 청킹 전략의 중요성 — 청크가 너무 크면 관련 없는 내용이 포함되고, 너무 작으면 맥락이 끊깁니다. 도메인에 맞는 구분자(헤더, 코드 블록)를 기준으로 청킹하는 것이 가장 효과적이었습니다.
  • 환각(Hallucination) 방지 — RAG로 사실 기반 답변을 유도하고, 프롬프트에 "모르면 모른다고 답해라"를 명시했습니다. 검색된 문서 없이 LLM만으로 답변하는 경우를 사전에 차단했습니다.
