Overview

React 19 + TypeScript로 Clausion 프론트엔드 구축

Clausion(AI 기반 교육 플랫폼)의 프론트엔드를 React 19와 TypeScript 5 strict 모드로 구축했습니다. 핵심 설계 원칙은 서버 상태(TanStack Query)와 클라이언트 상태(Zustand)를 명확히 분리하는 것이었습니다. 서버에서 오는 데이터는 캐싱·재요청·낙관적 업데이트가 필요하고, 클라이언트 전용 상태(인증 정보, UI 토글)는 별도 스토어로 관리하는 구조를 정착시켰습니다. React 19의 새로운 훅(use, useOptimistic, useFormStatus)과 Server Actions를 실험적으로 적용하며 최신 생태계 변화를 직접 체험했습니다.

Core Topics

핵심 학습 주제

🔷
TypeScript — 인터페이스·타입으로 런타임 오류 사전 방지
strict 모드를 활성화해 implicit any, null 가능성, 불필요한 타입 단언을 컴파일 단계에서 차단했습니다. API 응답 타입을 Zod로 런타임 검증하고, 제네릭을 활용한 커스텀 훅으로 타입 추론이 깨지지 않는 구조를 설계했습니다. 타입 가드(is, in, instanceof)와 discriminated union으로 상태에 따른 UI 분기를 안전하게 처리했습니다.
🌊
TanStack Query v5 — 서버 상태 관리
useQuery, useMutation, useInfiniteQuery로 서버 데이터의 페칭·캐싱·동기화를 선언적으로 처리했습니다. staleTime과 gcTime을 도메인별로 다르게 설정하고(강의 목록 5분, 사용자 정보 10분), 낙관적 업데이트(useOptimistic)로 UX를 개선했습니다. Suspense 모드와 Error Boundary를 결합한 로딩/오류 처리 패턴을 정착시켰습니다.
🐻
Zustand — 경량 전역 상태 관리
Redux 대비 보일러플레이트를 90% 이상 줄이면서도 미들웨어(devtools, persist, immer)를 조합해 충분한 기능을 확보했습니다. Slice 패턴으로 스토어를 도메인별로 분리하고, subscribe로 상태 변화를 외부 로직과 연동했습니다. Jotai와의 장단점을 비교 검토한 후 팀 컨벤션에 Zustand를 선택했습니다.
💨
TailwindCSS 4.2.2 — 유틸리티 기반 UI
CSS-in-JS 대비 런타임 오버헤드가 없고, 클래스 이름만 보면 스타일을 바로 파악할 수 있습니다. @layer components로 반복 패턴을 컴포넌트 클래스로 추출하고, arbitrary value([color:#a3e635])로 디자인 시스템 외 예외 케이스를 처리했습니다. Tailwind Merge + clsx 조합으로 동적 클래스 충돌을 방지했습니다.
⚡
Vite 6 — 빠른 개발 환경과 최적화 빌드
esbuild 기반 HMR이 50ms 이내로 반영되어 개발 생산성이 크게 향상됩니다. 프로덕션 빌드에서 Rollup의 tree-shaking과 code-splitting을 활용해 초기 번들 크기를 48% 줄였습니다. vite-plugin-pwa로 오프라인 지원과 캐싱 전략을 간단히 구성했습니다.
🎬
Framer Motion 12 — 선언적 애니메이션
페이지 전환, 목록 항목 등장, 모달 오버레이 등 UI 모션을 선언적으로 정의했습니다. AnimatePresence로 언마운트 애니메이션을 처리하고, useScroll + useTransform으로 스크롤 연동 시차(Parallax) 효과를 구현했습니다. layout prop으로 DOM 변화 시 자동 보간 애니메이션을 적용했습니다.

Real-World Application

실제 적용 사례

📚 Clausion — 강의 목록 & 진도 대시보드

TanStack Query로 강의 목록을 페이지네이션·캐싱 처리. useInfiniteQuery + 가상 스크롤(TanStack Virtual)로 수백 개의 강의도 부드럽게 렌더링했습니다. 학습 진도는 Zustand + localStorage로 오프라인 상태에서도 유지됩니다.

💬 실시간 채팅 UI

WebSocket 연결 상태를 Zustand로 관리하고, 수신 메시지를 TanStack Query 캐시에 직접 삽입(setQueryData)하여 서버 재요청 없이 실시간으로 화면을 갱신했습니다. Framer Motion으로 메시지 등장 애니메이션을 구현했습니다.

Lessons Learned

개발하며 배운 것들

  • 서버/클라이언트 상태 경계 — 처음엔 Zustand에 모든 상태를 넣었다가, API 데이터를 TanStack Query로 이전하면서 캐싱과 동기화 문제가 자동으로 해결되는 경험을 했습니다. 경계를 명확히 하는 것이 가장 중요합니다.
  • TypeScript strict 모드 활용 — strictNullChecks로 인한 에러가 처음엔 불편했지만, null 역참조로 인한 런타임 에러가 거의 사라졌습니다. 타입을 설계하는 시간이 디버깅 시간을 줄여줍니다.
  • QueryKey 설계의 중요성 — queryKey를 배열로 계층화해야 관련 쿼리를 일괄 무효화할 수 있습니다. 처음부터 체계적으로 설계하지 않으면 캐시 정합성 문제가 발생합니다.
  • 컴포넌트 분리 기준 — 재사용성보다 변경 이유가 같은 코드를 묶는 것이 올바른 분리 기준임을 경험했습니다. 너무 이른 추상화는 오히려 유지보수를 어렵게 만듭니다.
